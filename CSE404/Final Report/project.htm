A Brief Description

This is our Digital Systems Design course sessional and as part of the class, we have designed and implemented a simple game Space Fights.

The game uses Flex sensor and Sonar sensor to get user interaction and act accordingly.





Required Equipments

  • Arduino Macro

  • Sonar Sensor

  • Flex Sensor

  • 4X16 Decoders

  • 8X8 LED Matrix

  • Buzzer, Nano sound box





How it works?

We used Arduino Macro for interfacing other hardware peripherals. The whole project is basically an embedded system, which handles the interactions between the inputs and the outputs. The Arduino Mega is interfaced with 2 Flex Sensors as well as 2 Sonar Sensors. The four 8x8 Bi-color LED matrix are intefaced with Arduino Mega through 2 74HC154 4-to-16 decoders.

The Arduino continuously reads the inputs from Flexs and Sonars associated. The Sonars are used to move the player depending on the distance of the Sonar from a fixed reference. The Flexs are used to shoot. Certain thresholds are used in both sensors while reading inputs to resolve the issue of noisy data. When player shoots, the buzzer produce noises accordingly.
The Game has two modes: Single player and Two player. We can choose in which mode we want to play in the beginning of the game.

Problems Faced

    Sonar Sensor interfacing:

  1. Sonar Sensor calibration was difficult. We mapped sonar sensor distance with the column numbers on the 16x16 LED matrix board.

  2. Sonar sensor was causing some problem, i.e. giving 0 as output for no reason, we needed to fix that too.

  3. The most difficult part was when the sonar sensor giving too different values at the same physical distance which was resulting in vibration of the player's ship. For example, if I stop my hand at a particular position, it was giving 15cm for 3-4 time and 16 cm for 7-8 time. We had to handle this problem intellegntly.

  4. To measure the distance from sonar sensor a box like object was giving much better value. So we used a small box rather than human hand, and the sonar sensor had better accuracy.

  6. Once we declared many global variables which resulted into heap memory overflow. The code wad not uploading into the arduino. Then we optimized space by making most of them local variables

    7. Led matrix interfacing:

  7. For 4 led matrix we needed 24*4 = 96 pins alone ! This much pin was not available in arduino. So we had to use decoder. In this case we used 4x16 low logic decoder. For low logic we had to multiplex the columns of the leds
